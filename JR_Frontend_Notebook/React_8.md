# 重点总结

1. Readable, Maintainable, Reusable
2. SOLID
3. Component, Declarative
4. React 哲学
5. 肌肉记忆
  - 责任划分组件
  - 组件名称重复就有可能复用
  - UI 变化对应着 state
  - API 请求对应着 effect
  - 父传子，Props，子传父 State Lifting， 兄弟相传 State lifting
6. 不要 COPY PASTE
7. 不要 IF ELSE
8. 不要过度设计
9. 就近维护原则
